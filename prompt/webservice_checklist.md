# ✅ Webサービス公開前のチェックリスト

## セキュリティ

### 認証に関わるCookieの属性

- [ ] `HttpOnly` 属性が設定されていること（XSSの緩和策）
- [ ] `SameSite` 属性が `Lax` もしくは `Strict` になっていること（主にCSRF対策）
  - `Lax` の場合、GETリクエストで更新処理を行っているエンドポイントがないか合わせて確認
- [ ] `Secure` 属性が設定されていること（HTTPS通信でのみCookieが送られるように）
- [ ] `Domain` 属性が適切に設定されていること
  - サブドメインにもCookieが送られる設定の場合、他のサブドメインの脆弱性リスクを理解しておく

### ユーザーの入力値のバリデーション

- [ ] バリデーションがクライアント側だけでなく、サーバーサイド側でも行われていること
- [ ] ユーザー入力のURLのバリデーションが適切に行われていること
  - `javascript:` のようなURLを指定できないようにする（protocolの制限）
  - 正規表現を使う場合、バイパスできないかチェックする
- [ ] 受け取ったHTMLをそのまま出力する部分において、危険な文字列が渡される可能性が排除されていること
  - `element.innerHtml = input` や `dangerouslySetInnerHtml` のような部分
- [ ] ユーザーの入力値を表示する際にエスケープやサニタイズを行っていること
- [ ] SQLインジェクションが起こりうるSQL文が存在しないこと
- [ ] ユーザーがURLに含まれるハンドルネーム等を指定できる場合、バリデーションが適切に行われていること
  - アプリケーションで使用しているパスと被りうる文字列は拒否する
  - アンダースコアから始まる文字列は拒否する
  - 数字だけの文字列は拒否する
  - `admin` や `contact` などリザーブドの文字列を登録できないようにしておく

### レスポンスヘッダ

- [ ] `Strict-Transport-Security` がレスポンスヘッダに指定されていること
- [ ] 各ページのレスポンスヘッダに `X-Frame-Options: "DENY"` もしくは `"SAMEORIGIN"` が付与されていること（クリックジャッキング対策）
- [ ] `X-Content-Type-Options: nosniff` が指定されていること

### その他セキュリティ

- [ ] 退会・メールアドレス変更などの重要な操作で、直前のログインを必須にしていること（XSS・セッションハイジャックの緩和策）
- [ ] ユーザーにより内容が変わるレスポンスがCDNやKVSにキャッシュされていないこと
- [ ] オブジェクトストレージのディレクトリページのURLが公開されていないこと
- [ ] クライアントから渡されたURLにリダイレクトする部分で、バリデーションが行われていること（オープンリダイレクト対策）
- [ ] 更新・削除の処理において、権限を持たないユーザーや未認証ユーザーがデータを更新できないこと
- [ ] SQLの `DELETE` や `UPDATE` 文で `WHERE` が適切に設定されていること
- [ ] ユーザーから渡された文字列をそのままレスポンスヘッダに含めていないこと
- [ ] サーバーで発生したエラーメッセージをそのままブラウザに表示していないこと
- [ ] ファイルのアップロード機能において、ファイル形式・サイズ・ファイル名などのバリデーションが行われていること
- [ ] DBの定期的なバックアップが有効になっていること
- [ ] オブジェクトストレージのバックアップが有効になっていること
- [ ] 利用するクラウドサービスのアカウントで二要素認証が有効になっていること
- [ ] （サービスの要件次第で）CSPの設定がされていること

## ログイン

- [ ] メールアドレスの本人確認が行われていること
  - IDプロバイダから渡されたメールアドレスであっても本人確認ができているか検証すること
- [ ] 登録されているメールアドレスの列挙ができないこと（ログイン画面・パスワード再設定画面のエラーメッセージに注意）
- [ ] 複数のログイン方法を提供している場合、同一ユーザーがアカウント登録したときの仕様が決まっており、実装に反映されていること
- [ ] メールアドレス変更や、連携アカウントの変更ができること

---

## メール送信

- [ ] ユーザーの入力値がメールに含まれる場合、スパムメールが送られないこと
- [ ] ユーザーが特定の操作をしたときに不特定多数に繰り返しメールが送られないこと
- [ ] SPF / DKIM / DMARCの設定が完了していること
- [ ] バッチ処理でメールを送る場合、重複して送信されないこと（At least once delivery に注意）
- [ ] メルマガやキャンペーンメールの場合、ログインなしで購読解除ができるようになっていること
- [ ] 大人数にキャンペーンメールを送る場合、`List-Unsubscribe=One-Click` に対応していること

---

## SEO

- [ ] 全ページで `<title>` タグが適切に指定されていること
- [ ] SEO上重要になるページで canonical URLの設定がされていること
- [ ] エラー関連のページのステータスコードが `40x` もしくは `50x` になっている、もしくは `noindex` になっていること
- [ ] 検索結果ページに `noindex` もしくは canonical URLが設定されていること
- [ ] サイト全体に `noindex` が付与されていないこと（リリース時に解除するのを忘れがち）
- [ ] トップページなど検索流入が多くなるであろうページに `meta description` が設定されていること
- [ ] 動的にパブリックなページが生成される場合、XMLサイトマップを作成し、Search Consoleに登録されていること

---

## OGP

- [ ] 頻繁にシェアされることが想定されるページのOGP設定が完了していること
  - [ ] `og:title`
  - [ ] `og:description`
  - [ ] `og:url`
  - [ ] `og:image`
  - [ ] `twitter:card`（Xのカードの形式）

---

## 決済機能をつける場合

- [ ] どのように会計処理を行うか担当者と確認が取れていること
- [ ] 決済に失敗したときに、アプリケーション上のデータと決済代行サービス上のデータで不整合が生じないこと
- [ ] 万が一不整合が生じた場合に検知できるようになっているか
- [ ] 重複決済が発生しない実装になっていること
- [ ] 決済を行ったことがあるユーザーが退会しても会計やアプリケーションのロジックに不整合が生じないこと
- [ ] サブスクリプションを契約中のユーザーが退会（もしくはアカウント凍結）したときに、サブスクリプションが自動キャンセルされること
- [ ] 決済を行ったことがあるユーザーが退会したときの返金有無や日割り計算について利用規約に書かれていること
- [ ] 解約の導線が用意されていること
- [ ] カードの期限切れ等によりサブスクリプションの更新が失敗したときのハンドリングができていること
- [ ] 領収書が適格請求書の要件を満たしていること（インボイス制度）

---

## アクセシビリティ

- [ ] 画像（`<img>`）の `alt` 属性が適切に指定されていること
- [ ] 内部がSVGアイコンだけの `<button>` や `<a>` の役割がスクリーンリーダーからも認識できるようになっていること（`aria-label` などを使用）

---

## パフォーマンス

- [ ] 余計なモジュールがバンドルJSに含まれていないこと（bundle-analyzerなどで確認）
- [ ] 静的ファイルがCDNにキャッシュされていること
- [ ] 画像によるレイアウトシフトが起きないこと（`aspect-ratio` や `width/height` 属性を指定）
- [ ] 必要以上に巨大な画像が読み込まれていないこと
- [ ] SQLのインデックスが適切に貼られていること

---

## 複数環境での動作確認

- [ ] スマホやタブレットサイズの画面で表示したときにUIが崩れないこと
- [ ] 各OS・ブラウザでフォントがおかしなことになっていないこと（`font-family` がMac/Windows/iOS/Androidで不自然にならないか）
- [ ] ChromeだけでなくSafariやFirefoxでの動作確認もしていること
- [ ] （開発環境がMacの場合）スクロールバーを常に表示にしても問題がないこと
- [ ] ユーザーが指定したハンドルネームなどの入力値が長いときに見た目が崩れないこと

---

## その他

- [ ] ローカルストレージやhttp-onlyでないCookieが7日で消えても問題がないこと（iOS SafariのITP対策）
- [ ] サードパーティCookieに依存していないこと
- [ ] 日本語サイトの場合、`<html lang="ja">` となっていること
- [ ] サーバーエラーが発生したときにエラーの内容が通知される or 検知できるようになっていること
- [ ] 404ページや50x系ページが適切に表示され、次のアクションへの導線が表示されていること
- [ ] ファビコンが設定されていること
- [ ] `apple-touch-icon` が設置されていること
- [ ] Google Analyticsなどアクセス解析ツールを導入していること（必要なら）
- [ ] クローズドチャットなどのサービスを提供する場合、電気通信事業者の申請をしていること
- [ ] サービス名が他言語でおかしな意味でないこと
