このリポジトリの既存ウェブサイトを3人のエージェントチームでレビューし改善計画を `docs/[YYYYMMDD]-improvement-plan.md` に作成してください。

エージェントは
【Designer】UI/UX改善。ユーザーフロー・情報設計・インタラクションの課題を発見し改善案を提示
【A11y】WCAG 2.2 AA準拠の監査。Designerの改善案もレビュー
【Engineer】Nextjsベストプラクティス比較、実装面の技術負債・パフォーマンス・保守性を評価し、改善の工数をT-shirt sizingで見積もる

の3名です。発言時は必ずプレフィックスをつけてください。

進行はフェーズゲート制とします。
Phase 1: 各自が専門視点で現状を分析し、課題一覧を作成。
Phase 2: 課題を突き合わせて重複を統合し、優先度（高/中/低）をつける。優先度はユーザー影響度×改善コストで判定。
Phase 3: 高優先の課題から改善案を具体化し、相互レビュー
　　　　　（指摘は「必須/推奨/質問」で分類し、必須を全解消してから次へ）。
Phase 4: 品質チェック
 Designer: 改善後のフローが現状より手順削減されているか
 A11y: キーボード操作完結・色のみ依存・代替テキスト不備がないか
 Engineer: 各改善が見積もり付きで実行可能か

全パスしたら出力してください。

不足条件は仮定で補完し、文書の冒頭に仮定一覧（影響度: 高/中/低）として明記してください。
エージェント間で意見が対立した場合は「ユーザー価値への影響度」で優先判断し、合意できない場合は両論併記のうえ推奨案を示してください。

改善計画の構成は、
　仮定一覧
　現状サマリー
　課題一覧（優先度・カテゴリ付き）
　改善案（それぞれに担当観点・工数見積・期待効果を併記）
　アクセシビリティ監査結果
　改善ロードマップ（Quick Win / 短期 / 中期）
　リスクと未解決課題の順とします。

最後に各エージェントが使用したskillsを明記してください。

Phase 1から開始してください。
