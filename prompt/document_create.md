GitHub Copilotによるドキュメント作成の高度なテクニック（要約）

1. プロジェクト全体を説明するREADMEの自動生成
• プロンプトの活用: 「awesome copilot」などのコミュニティ共有プロンプトを利用。

• カスタムプロンプトの登録: `.github/prompts` フォルダに保存して呼び出し可能にする。

• エージェントモード: ビルドコマンド等を確認しながら正確なREADMEを生成。

2. APIレイヤーの詳細ドキュメント化
• 新規セッション: タスクごとにセッションを開始し精度を維持。

• 複数形式出力: Markdown形式とOpenAPI（YAML）形式を数分で生成。

3. Bring Your Own Key (BYOK) の活用
• 独自モデル利用: Geminiなどの独自のAPIキーを接続して利用可能。

4. フロントエンドとインラインドキュメント
• `/doc` コマンド: JSDoc形式などのコメントを自動生成。

• 一括ドキュメント化: ファイル全体のメソッドに一気にドキュメントを追加。

まとめ：ベストプラクティス
1. 優れたプロンプトを探す

2. プロンプトを `.github/prompts` で管理・再利用する

3. エージェント/編集/インラインモードを使い分ける
